## 🎯 为什么选择 JobGraph？

### 🔒 隐私保护 — 核心差异化

**在 Boss 直聘等平台求职时，你是否担心过：**

| 风险 | 说明 |
|------|------|
| 👀 **浏览记录被追踪** | 你看了哪些公司，HR 都知道 |
| 📊 **求职意向暴露** | 你的期望薪资、跳槽动机被分析 |
| 🏢 **现公司发现** | 在职看机会，可能被领导知道 |
| 📝 **简历被滥用** | 个人信息可能被倒卖 |

**JobGraph 解决方案：**

| 特性 | 说明 |
|------|------|
| 🏠 **私有部署** | 数据存在你自己的电脑/服务器 |
| 🔐 **本地运行** | 所有操作在本地完成，不上传任何数据 |
| 👻 **无痕浏览** | 查看公司、搜索岗位不留任何痕迹 |
| 🚫 **无需注册** | 不需要手机号、邮箱等个人信息 |
| 💾 **数据自控** | 你的数据只属于你 |

### 🎯 核心功能

| 功能 | 说明 |
|------|------|
| 🔍 **岗位搜索** | 多平台聚合，智能筛选 |
| 🏢 **公司画像** | 员工评价、风险评分 |
| ⚠️ **避坑预警** | 坑点识别、黑名单 |
| 📊 **薪资分析** | 市场行情、合理定价 |
| 🎯 **智能匹配** | 简历分析、个性化推荐 |

## 💡 使用场景

### 场景一：在职看机会
```
你在一家公司上班，想看看外面的机会
├── ❌ 用 Boss 直聘 → 可能被现公司 HR 发现
└── ✅ 用 JobGraph → 私密浏览，无人知晓
```

### 场景二：了解公司真实情况
```
拿到一个 Offer，想了解公司真实情况
├── ❌ 在脉脉搜 → 记录可能被关联
└── ✅ 用 JobGraph → 本地查询，完全私密
```

### 场景三：批量筛选公司
```
想系统性了解某个行业的公司
├── ❌ 在招聘网站搜 → 算法会推荐类似岗位给你的同事
└── ✅ 用 JobGraph → 自主分析，不受算法干扰
```

## 🚀 快速开始

### 方式一：本地调试 (推荐)

```bash
# 1. 克隆项目
git clone <repository-url>
cd multimodal-graph-rag

# 2. 一键启动 (自动安装依赖、启动 Neo4j)
./dev.sh

# 3. 导入示例数据
make jobgraph-import

# 4. 启动求职图谱
make jobgraph
# 访问 http://localhost:8504
```

### 方式二：Docker 部署

```bash
# 一键部署
./deploy.sh
```

## 📦 项目结构

```
multimodal-graph-rag/
├── src/
│   ├── jobgraph/           # 求职图谱核心模块
│   │   ├── models.py       # 数据模型
│   │   └── graph_manager.py # 图谱管理器
│   ├── graph/              # Neo4j 客户端
│   └── ingestion/          # 数据摄入
├── web/
│   └── jobgraph.py         # 求职图谱界面
├── scripts/
│   ├── import_jobs.py      # 导入示例数据
│   └── incremental.py      # 增量更新脚本
├── deploy.sh               # Docker 部署脚本
├── dev.sh                  # 本地调试脚本
└── Makefile                # 快捷命令
```
